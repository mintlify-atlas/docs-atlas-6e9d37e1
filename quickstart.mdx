This guide walks you through cloning the repository, installing dependencies, and running the full WASI test suite against an available runtime.

## Prerequisites

- **Git** — to clone the repository.
- **Python 3** and **pip** — the test runner is a Python script.
- At least one supported WASI runtime installed and on your `PATH` (for example, `wasmtime`, `wasmedge`, or `wazero`). The runner detects available runtimes automatically.

## Run the test suite

<Steps>
  <Step title="Clone the repository">
    Use the `prod/testsuite-base` branch. Unlike `main`, this branch includes precompiled `.wasm` test binaries so you do not need a C or Rust compiler toolchain.

    ```bash
    git clone --branch prod/testsuite-base https://github.com/WebAssembly/wasi-testsuite
    cd wasi-testsuite
    ```
  </Step>

  <Step title="Install Python dependencies">
    Install `python3` and `pip` if you do not have them. On Ubuntu/Debian:

    ```bash
    sudo apt install python3 python3-pip
    ```

    Then install the test runner's Python dependencies:

    ```bash
    python3 -m pip install -r test-runner/requirements.txt
    ```

    The runner depends on `colorama` (for colored terminal output) and `requests`.
  </Step>

  <Step title="Run the tests">
    Execute the test suite. By default, the runner detects every runtime it finds an adapter for in the `adapters/` directory and runs tests against all of them.

    ```bash
    ./run-tests
    ```

    The runner prints which runtimes it detected, then streams pass/fail results for each test case.
  </Step>
</Steps>

## Target a specific runtime

Pass `--runtime-adapter` (short flag: `-r`) with the path to an adapter file to run tests against a single runtime instead of all detected runtimes.

```bash
./run-tests --runtime-adapter adapters/wasmtime.py
```

Available adapters:

| Adapter file | Runtime |
|---|---|
| `adapters/wasmtime.py` | wasmtime |
| `adapters/wasmedge.py` | WasmEdge |
| `adapters/wasm-micro-runtime.py` | WAMR (iwasm) |
| `adapters/wazero.py` | wazero |
| `adapters/pywasm.py` | pywasm |
| `adapters/wizard.py` | Wizard |

## Override the runtime binary

Each adapter invokes a runtime binary by name (for example, `wasmtime`, `iwasm`). You can override the binary — or pass extra flags — by setting a corresponding environment variable.

<Tip>
  This is useful when you have multiple builds of a runtime installed, or when you need to enable experimental features for testing.
</Tip>

```bash
# Use a specific wasmtime build and enable all Wasm features
WASMTIME="wasmtime --wasm-features all" ./run-tests --runtime-adapter adapters/wasmtime.py

# Point to a custom iwasm binary
IWASM="/opt/custom-wamr/bin/iwasm" ./run-tests --runtime-adapter adapters/wasm-micro-runtime.py
```

The environment variable name matches the uppercase binary name the adapter normally calls (`WASMTIME`, `IWASM`, `WASMEDGE`, `WAZERO`, etc.).

## Skip known-failing tests

Use `--exclude-filter` to pass a JSON file listing tests to skip. An example filter file is provided in the repository:

```bash
./run-tests --exclude-filter examples/skip.json
```

You can pass `--exclude-filter` multiple times to layer several filter files.

## Save results as JSON

Pass `--json-output-location` to write structured test results to a file in addition to the console output:

```bash
./run-tests --json-output-location results.json
```
